## 3. PHASE 1: THE AI LABORATORY (Anaconda & Jupyter)
Before building APIs, the ML models must be trained. You will provide the Python code for `model_training.ipynb`.

**Workflow:**
1.  **Environment Setup:** Note that the user will run `conda create -n energy_ai python=3.11` and `conda activate energy_ai`.
2.  **Data Generation:** Inside the notebook, write a script to generate a synthetic dataset (1 month of hourly data) for Flat "A101".
    *   Base load: 0.5 kWh. Morning peak: 2.5 kWh. Evening peak: 4.0 kWh.
    *   Inject 3% random anomalies (10.0 - 15.0 kWh).
3.  **Feature Engineering (Cyclical Encoding):** Convert timestamps into cyclical features to map time properly:
    *   `hour_sin = np.sin(2 * np.pi * hour/24)`, `hour_cos = np.cos(2 * np.pi * hour/24)`
4.  **Model Training:**
    *   Train `IsolationForest(contamination=0.05)` on cyclical features + units.
    *   Train `LinearRegression()` on cyclical features to predict units.
5.  **Export:** Use `joblib` to save the trained models into the `/models/` folder as `iso_model.pkl` and `lr_model.pkl`.

## 4. PHASE 2: ML INFERENCE SERVICE (FastAPI)
Once the `.pkl` files exist, FastAPI will use them for real-time inference.

*   **POST `/api/ml/anomaly`**
    *   **Input Schema:** `{"units": float, "timestamp": str}`
    *   **Logic:** Load `iso_model.pkl`. Parse timestamp to extract hour/day, apply cyclical encoding. Run model prediction.
    *   **Output:** `{"is_anomaly": bool}`
*   **POST `/api/ml/predict`**
    *   **Input Schema:** `{"target_timestamp": str}`
    *   **Logic:** Load `lr_model.pkl`. Apply cyclical encoding to the target timestamp. Run prediction.
    *   **Output:** `{"predicted_units_kWh": float}`

## 5. PHASE 3: BACKEND SYSTEM (Node.js/Express)

### A. Database Schema (`/models/EnergyData.js`)
*   `flatId` (String, required, default: "A101")
*   `timestamp` (Date, required, default: Date.now)
*   `units_kWh` (Number, required)
*   `isAnomaly` (Boolean, default: false)
*   **Index:** Compound index `{ flatId: 1, timestamp: -1 }`.

### B. Core APIs (`/controllers/energyController.js`)
*   **POST `/api/energy/ingest`**: 
    1. Receives payload: `{ flatId, units }`. Generates current timestamp.
    2. Makes an Axios POST call to FastAPI `/api/ml/anomaly`.
    3. **Resilience logic:** Wrap in try/catch. If FastAPI fails/timeouts, default `isAnomaly` to `false`. Do NOT crash.
    4. Save to MongoDB. Return 201.
*   **GET `/api/energy/history/:flatId`**: Fetch the last 50 records sorted by timestamp descending, return them ascending (for frontend charts).
*   **GET `/api/energy/predict/:flatId`**: Calculate timestamp for `current hour + 1`. Call FastAPI `/predict`. Return the value.

### C. The IoT Simulator (`simulator.js`)
*   Standalone script using `setInterval` (runs every 5 seconds).
*   Generates a normal unit reading based on current local time.
*   Has a 5% chance to spike the reading (simulate malfunction).
*   POSTs the reading to `/api/energy/ingest`. Console logs success/failure in colors.
